# Azure Disk and Snapshot Cost Optimizer
**Author:** [Satish Balakrishnan](mailto:satishbal@microsoft.com)  
## Script

`02_azure_disk_snapshot_optimizer_v1.sh`

This optimizer analyzes managed disks and snapshots using the baseline cost report generated by the core cost collector. It identifies cost optimization opportunities such as unattached disks, older snapshots, and possible disk SKU or tier optimization.

---

## Prerequisites

Before running this script, run the core collector first and generate:

```text
FinalRankedReport.csv
```

The disk and snapshot optimizer requires `FinalRankedReport.csv` as its main input.

Example core collector flow:

```bash
cd ../../core
chmod +x 00_azure_cost_data_collector_v1.sh
./00_azure_cost_data_collector_v1.sh subs.txt FinalRankedReport.csv
```

The script also requires:

- Azure CLI
- Python 3
- Azure login completed
- Reader access to target subscriptions
- Permission to read disk and snapshot metadata
- Permission to read Azure Monitor metrics where disk metrics are used

---

## How to Run

From the `optimizers/disks/` folder:

```bash
chmod +x 02_azure_disk_snapshot_optimizer_v1.sh
./02_azure_disk_snapshot_optimizer_v1.sh ../../core/FinalRankedReport.csv Disk_Snapshot_Optimization_Output.csv
```

Optional example for top results:

```bash
TOP_N=5 ./02_azure_disk_snapshot_optimizer_v1.sh ../../core/FinalRankedReport.csv Disk_Snapshot_Optimization_Output_top5.csv
```

Optional snapshot age threshold:

```bash
SNAPSHOT_OLD_DAYS=90 ./02_azure_disk_snapshot_optimizer_v1.sh ../../core/FinalRankedReport.csv Disk_Snapshot_Optimization_Output.csv
```

---

## Output

```text
Disk_Snapshot_Optimization_Output.csv
```

The output includes disk and snapshot cost, attachment state, SKU or tier, size, age, metrics, estimated savings, and recommendation confidence.

---

## Notes

This script is read-only. It does not delete disks, delete snapshots, change disk tiers, detach disks, or modify any Azure resources.
